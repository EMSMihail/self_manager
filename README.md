# Self Manager v1.0.0

Персональный менеджер задач и заметок с Kanban-доской, интеграцией с Telegram-ботом для уведомлений и кастомизацией интерфейса, а также интеграцией со следующими api:
- Unsplash

## 📁 Структура проекта

```text
.
├── Dockerfile              # Multi-stage сборка (Svelte статика -> Go бинарник)
├── docker-compose.yml      # Конфигурация для деплоя и запуска
├── tg_backup.sh            # Скрипт резервного копирования данных из бд в Telegram бота
├── backend/                # Бэкенд-приложение на Go
│   ├── main.go             # Точка входа, инициализация роутов и раздача статики
│   ├── bot/                # Модуль Telegram-бота (воркер уведомлений)
│   ├── db/                 # Слой работы с бд (SQLite [драйвер CGO])
│   ├── models/             # Структуры данных и модели (note.go)
│   └── handlers/           # Обработчики API-запросов
└── frontend/               # Фронтенд-приложение на Svelte
    ├── src/                # Исходный код интерфейса
    │   ├── lib/            # Общие компоненты, API-клиент и IndexedDB
    │   └── routes/         # Страницы и маршрутизация SvelteKit
    ├── static/             # Статические ресурсы (иконки, robots.txt)
    ├── vite.config.js      # Конфигурация сборщика Vite
    └── package.json        # Зависимости и скрипты Node.js
```
## 🛠 Переменные окружения

Хранятся в файле `backend/.env`:

```
TELEGRAM_TOKEN=your_telegram_bot_token
TELEGRAM_CHAT_ID=your_personal_chat_id
UNSPLASH_ACCESS_KEY=your_unsplash_api_key
```

## Prerequisites
Для локального запуска/отладки/разработки (без докера) убедитесь, что у вас установлены:
- Golang (1.22+)
- NodeJS (20+)

Во всех иных случаях:
- Docker (Engine, 20+)

## Шпаргалка по командам

### Бэкэнд
```
cd backend
go mod download
go run main.go
```
В случае, если захотите заново проинициировать проект и пойти со своими модулями
`go mod tidy` - для проверки соответствия go.mod файла и коду бэкэнда

### Фронтэнд
```
cd frontend
npm install
npm run dev
```
Аналогичная бэкэнду ситуаиця

### Контейнер

`docker-compose build && docker-compose up -d` - Сборка и запуск приложения в компознике
`docker-compose up --build` - Быстрая команда для запуска и получения логов
`docker-compose build --no-cache` - В случае проблем из-за закешированных слоёв и отсутсвия изменений, можно попробовать пересобрать
`docker compose logs -f`  - Команда для просмотра логов (P.S. Если у вас много различных контейнеров в композе запущено, нужно уточнять имя запущенного приложения)
